# 拆书工作区

基于 claude-obsidian wiki 知识库的拆书内容生产工作区。

## 目录结构

```
拆书/
├── 选题/          # 热门书单 + 拆书计划（book-trending 输出）
├── 输出/          # 拆书内容输出（book-cracker 输出）
│   └── {书名}_拆书输出.md   # 四平台合一：小红书 + 知乎 + 头条 + 视频脚本
├── 视频/          # 视频生成（book-cracker-video 输出）
│   ├── {书名}.mp4            # 最终竖屏视频 1080x1920
│   └── temp/                 # 临时文件（可删除）
└── bgm/           # 背景音乐素材（MP3/WAV/M4A）
```

## 工作流

```
                  book-trending
                      |
                      v
                  选题/ (选书)
                      |
                      v
.raw/*.epub ---> wiki-ingest ---> wiki/sources/ (知识沉淀)
                      |                |
                      v                v
                  book-cracker (拆书)
                      |
                      v
                  输出/{书名}_拆书输出.md
                      |
                      v
                  book-cracker-video
                      |
                      v
                  视频/{书名}.mp4
```

## 与 wiki 的关系

1. **知识来源**：电子书原文放在 `.raw/`，通过 `wiki-ingest` 导入 wiki 知识库
2. **知识沉淀**：`wiki/sources/` 存放每本书的结构化知识笔记
3. **内容生产**：`book-cracker` 基于 wiki 知识 + 原书生成四平台内容
4. **视频生产**：`book-cracker-video` 消费拆书输出中的视频脚本，生成竖屏 MP4

## 命名规范

- 拆书输出：`{书名}_拆书输出.md`（如 `认知觉醒_拆书输出.md`）
- 视频输出：`{书名}.mp4`（如 `认知觉醒.mp4`）
- 选题书单：`trending-{方向}-{YYYY-MM-DD}.md`

## 拆书输出文件结构

每个 `{书名}_拆书输出.md` 包含：

1. **拆书卡片**：书名片 + 核心观点 + 逻辑脉络 + 金句Top-8 + 切入角度
2. **小红书笔记**：3分钟速读卡片（300-450字，emoji视觉锚点）
3. **知乎回答**：深度拆书论证（1500-3000字，blockquote引用）
4. **头条文章**：3分钟速读短文（500-800字，含配图生图提示词）
5. **视频脚本**：纯净口播正文 + 导演标注版 + BGM情绪 + 关键文字标记 + 分段关键词
